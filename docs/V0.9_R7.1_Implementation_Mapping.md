## Current System Model

- The workflow lock is the cross-process mutation boundary.
- RuntimeState is not resource truth; the Recovery Journal and observed process/display facts are required.
- Clean Normal authorizes a new Enable and final Restore only. It does not describe healthy active Headless operation.
- Managed process cleanup requires independently verified executable, file identity and start identity.
- Future persistence schemas are preserved unchanged.
- Uninstall is a destructive consumer of recovery tools and therefore must fail closed on every unknown fact.
