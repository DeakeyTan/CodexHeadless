# V0.9 R3 Fallback Read Audit

| Location | Classification | Allowed |
|---|---|---|
| App status/menu/dialog `load()` calls | Read-only presentation or preflight; Core revalidates every side effect | Yes |
| CLI post-operation/status `load()` calls | Output only; operation itself uses throwing Core APIs | Yes |
| `StatusSnapshotProvider` and `Doctor` | Read-only diagnostics; degraded state is displayed | Yes |
| `RuntimeWorkflowCoordinator.refreshPhaseIfNeeded` | Best-effort transition from expired cooldown text to idle; no resource/display decision | Yes |
| `VirtualDisplayManager.waitForNewDisplayID` best-effort phase update | Presentation-only phase/deadline while resource ownership remains in Journal | Yes |
| Enable/Restore/Recovery/Sleep safety decisions | Throwing `read()` / `transaction()` required | Enforced |
| Virtual lifecycle and ownership decisions | Throwing `read()` / Journal read required; only phase text is best effort | Enforced |

`scripts/audit_critical_reads.sh` rejects new fallback reads in critical workflow files.
