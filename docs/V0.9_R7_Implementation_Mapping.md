## Current Lifecycle Model

- Workflow lock is the cross-process mutation boundary; Enable reassesses Clean Normal after lock acquisition.
- Recovery Journal is durable workflow authority and future schemas are preserved.
- Managed resources require intent, observed ownership, Journal persistence, RuntimeState commit, and independently verified cleanup.
- R6 virtual helper consumes an exact-stage one-time capability, waits for exact parent continuation, and reports an exact ready display ID.
- Restore success requires physical takeover, resource disappearance, clean RuntimeState and Journal finalization.
- App cache is presentation-only; it must expose classification without weakening Core authority.
