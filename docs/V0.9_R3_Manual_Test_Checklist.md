# V0.9 R3 Manual Test Checklist

Use a Mac with SSH recovery access and at least one test configuration with an external/dummy display. Do not use production-only data without a recovery path.

## Brightness

- [ ] NOT VERIFIED: With no reliable brightness read capability and a configuration that may need brightness fallback, Enable refuses before Keep Awake or virtual-display creation.
- [ ] NOT VERIFIED: No brightness key events are emitted and no fixed `0.7` restore occurs.
- [ ] NOT VERIFIED: If verified brightness restoration fails during Restore, the last replacement display and Keep Awake remain available.

## Keep Awake

- [ ] NOT VERIFIED: Enable creates one CodexHeadless Keep Awake helper.
- [ ] NOT VERIFIED: `pmset -g assertions` identifies that helper PID as the assertion holder.
- [ ] NOT VERIFIED: Restore removes the assertion and only then reports Keep Awake Off.
- [ ] NOT VERIFIED: Reused/mismatched PID is preserved and produces Recovery Required rather than termination.

## Virtual Display

- [ ] NOT VERIFIED: Enable creates and enumerates the managed virtual display.
- [ ] NOT VERIFIED: Restore first restores/promotes a physical display, then stops the host, then observes managed display disappearance.
- [ ] NOT VERIFIED: Killing or replacing the recorded host produces ownership drift and does not kill an unrelated process.

## Journal And Reentry

- [ ] NOT VERIFIED: Force failure after resource start but before RuntimeState commit; Journal remains and Restore can resume after App/CLI restart.
- [ ] NOT VERIFIED: Force terminate failure with the resource still observed; RuntimeState is not Normal and resource is not reported Off.
- [ ] NOT VERIFIED: A future-schema Journal remains byte-identical after status, Doctor, Enable, and Restore attempts.
- [ ] NOT VERIFIED: Interrupt Restore at each cleanup phase and confirm the next Restore resumes from independently verified progress.

## Clean Normal And App

- [ ] NOT VERIFIED: Every RuntimeState, Journal, observed-owner, managed-display, brightness, soft-disconnect, and Touch Bar violation blocks Enable.
- [ ] NOT VERIFIED: App configuration controls are disabled while Preparing, Confirm Required, Headless, Restoring, Error, or Recovery Required.
- [ ] NOT VERIFIED: `status` and `doctor` agree on Clean Normal and show recorded/observed ownership plus cleanup progress.

## Helper Authorization

- [ ] NOT VERIFIED: Direct legacy underscore helper commands return nonzero without side effects.
- [ ] NOT VERIFIED: Missing, expired, replayed, wrong-parent, wrong-kind, wrong-operation, and wrong-executable capabilities are rejected.

## Release Installation

- [ ] NOT VERIFIED: Install the unsigned universal PKG on Apple Silicon macOS 13+.
- [ ] NOT VERIFIED: Install the same PKG on Intel macOS 13+.
- [ ] NOT VERIFIED: Launch the menu bar app, run CLI status/doctor, Enable, Confirm, Restore, and verify a second full cycle.
- [ ] NOT VERIFIED: Confirm icon, hotkeys, Touch Bar restoration, display layout restoration, and recovery after app restart.

Manual items remain `NOT VERIFIED`; no item is implied by build or parser success.
