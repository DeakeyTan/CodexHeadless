## Completed Changes

- Replacement continuity extends existing periodic Operational Evidence. The exact replacement ID is assessed; one trusted miss creates an in-memory suspect; one bounded 1.5-second confirmation either clears the suspect or submits the existing App Restore wrapper exactly once.
- Critical evidence now requires RuntimeState Keep Awake intent, exact managed virtual ID/type, exact physical replacement facts, and the existing built-in soft-disconnect direction.
- Operational Evidence cache use is mode/operation compatible. New Enable and every verified-Normal App path retire old evidence. Automatic rollback uses the shared verified-Normal cleanup. Previous transition presentation uses previous evidence mode.
- LaunchAgent verification is three-state and fail-closed. Only recognized service-not-found permits deletion; unknown results preserve App and CLI while the uninstall workflow lock remains held.
- Persistent diagnostics are default-disabled and controlled by existing App/CLI configuration paths. One shared in-memory policy gates `CHLogger` before file work and propagates through the existing helper environment.

## Required Proof

**Default logging disabled:** `AppConfig.default` and missing optional values resolve false. App and ordinary CLI force policy off before bootstrap config load. Disabled tests prove no log file, lock file, rotation, write, or fsync path is reached.

**User-visible errors preserved:** the switch gates only `CHLogger`; CLI `print`/exit, App alerts, menu presentation, helper protocol stdout/stderr, workflow decisions and Restore results remain outside the policy. Existing authorization, presentation and workflow suites pass.

**Existing Restore reused:** automatic replacement recovery calls `runRestoreControllerOperation`, whose operation is `controller.restoreNormal()`. It does not directly enable a display, kill a host, clear RuntimeState, or bypass takeover/Journal cleanup. A failed coordinator submission clears suppression so a later periodic observation can retry.

**Cache cannot cross workflows:** cache availability checks mode and expected operation ID; new Enable clears history; every verified Normal clears evidence and expected operation identity. Incompatible refresh history yields unavailable/refreshing-without-history, not healthy or Recovery Required.

**Unknown LaunchAgent preserves tools:** timeout, signal, execution exception, unexpected/empty output and domain-style failures classify as unverified. `stopAndVerify` returns before coordinator deletion, and focused tests assert the deleter remains untouched.
