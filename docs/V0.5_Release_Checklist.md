# CodexHeadless v0.5 Release Checklist

This checklist captures the verified v0.5 workflow for the target MacBook Pro.

## Recommended Configuration

```bash
codex-headless config set resolution 2560x1440
codex-headless config set scale-mode hidpi
codex-headless config set virtual-display-policy always
codex-headless config set soft-disconnect on
codex-headless config set touchbar-hide on
codex-headless config set keep-awake-backend caffeinate
codex-headless config set hotkeys.enabled true
codex-headless config set confirm-dialog.enabled true
```

## Preflight

```bash
./scripts/install.sh
codex-headless self-test
codex-headless doctor
codex-headless virtual-display probe
codex-headless soft-disconnect probe
codex-headless soft-disconnect check
codex-headless touchbar check
codex-headless config get hotkeys
codex-headless config get confirm-dialog
```

Expected:

- CLI is installed at `/usr/local/bin/codex-headless`.
- App bundle is installed at `/Applications/CodexHeadless.app`.
- Self test passes.
- Doctor sees the external / HDMI Dummy display.
- `virtual-display-policy` is `always` or `auto`.
- Soft-disconnect check is allowed when the external / Dummy display is main.
- Touch Bar check is allowed when `touchbar-hide=on`.
- Hotkeys are enabled with `⌃⌥⌘⇧E`, `⌃⌥⌘⇧C`, and `⌃⌥⌘⇧R`.
- Confirm dialog is enabled.

## Headless Workflow

```bash
codex-headless on
codex-headless confirm
codex-headless status
codex-headless off
```

Expected after `confirm`:

- Mode is `Headless`.
- Keep Awake is `On`.
- External / Dummy display remains `Main: Yes`.
- Managed Virtual display is active when policy is `always`.
- Built-in display is soft-disconnected and absent from the display list.
- Touch Bar state is hidden when `touchbar-hide=on`.
- Rollback guard is confirmed.

Expected after `off`:

- Mode is `Normal`.
- Keep Awake is `Off`.
- Caffeinate PID is not running.
- Built-in display appears again.
- Managed Virtual display is stopped.
- Touch Bar state is active / not hidden.

## Status Bar App

Open:

```bash
open /Applications/CodexHeadless.app
```

Expected:

- Menu bar shows a `CH` item.
- The menu exposes mode, Keep Awake, virtual display, built-in handling, and Touch Bar state.
- `Apply Recommended v0.5 Config` sets the verified v0.5 defaults.
- `Virtual Display` menu exposes policy, scale mode, presets, and custom resolution.
- `Display & Touch Bar Safety` menu exposes soft-disconnect and Touch Bar hide toggles.
- `Diagnostics` can copy status, doctor, and self-test reports.
- `Hotkeys` menu shows registration status for Enable, Confirm, and Restore.
- `Confirm Dialog` menu can enable or disable the floating confirmation panel.
- App bundle has a custom icon generated from `icon.png`.

## Confirm Dialog And Hotkeys

Expected:

- Menu bar Enable shows the Confirm / Rollback panel after entering `Confirm Required`.
- Panel Confirm enters `Headless` and closes the panel.
- Panel Rollback restores `Normal` and closes the panel.
- Closing the panel does not confirm; rollback guard remains active.
- `⌃⌥⌘⇧E` enables Headless only from `Normal`.
- `⌃⌥⌘⇧C` confirms only from `Confirm Required`.
- `⌃⌥⌘⇧R` restores from `Preparing`, `Confirm Required`, `Headless`, `Fallback`, or `Error`.
- CLI `codex-headless confirm` or `codex-headless off` closes the panel while the app is running.

## Safety Notes

- `pmset` is best-effort only. It uses non-interactive `sudo -n`; missing sudo credentials should be logged and skipped, not block `on`.
- Private display and Touch Bar APIs must stay behind isolated helper subprocesses.
- Never soft-disconnect the only active display.
- Keep SSH access available before testing new display variants.
