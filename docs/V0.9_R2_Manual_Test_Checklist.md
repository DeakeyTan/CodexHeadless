## Apple Silicon

- [ ] No external display: Enable creates managed virtual display, confirms, and restores built-in display before virtual cleanup.
- [ ] External display: external remains main; managed virtual policy behaves as configured.
- [ ] Soft-disconnect: built-in disappears and Restore reconnects it from journal ID.
- [ ] Brightness fallback: permission preflight blocks before handoff when Accessibility is absent.
- [ ] Confirm and manual rollback.
- [ ] Automatic rollback with healthy and damaged config.
- [ ] RestorePaused: replacement and Keep Awake remain active; attach physical display and resume.
- [ ] Corrupt state with healthy journal.
- [ ] Corrupt config; corrupt state plus config.
- [ ] Corrupt/missing journal: Recovery Required, no unverifiable kill.
- [ ] Virtual PID gone while display remains: cleanup incomplete/drift.
- [ ] Keep Awake PID reuse/start-time mismatch: cleanup refused and state remains On.
- [ ] App Enable then CLI Restore; CLI Enable then App Restore.
- [ ] Restart App during cleanup and continue Restore.
- [ ] Menu instance remains stable while countdown/status updates.
- [ ] Menu Quit, Command-Q, Dock Quit, and system termination are blocked with active resources.
- [ ] Install/uninstall unsigned PKG and launch embedded/standalone CLI.
