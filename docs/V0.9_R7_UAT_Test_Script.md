# CodexHeadless 0.9.0-uat.1 UAT Test Script

For every case record timestamps, final mode/classification, relevant log lines, and whether Restore succeeded.

| Case | Preparation and action | Expected result | Recovery if different |
|---|---|---|---|
| Launch/status | Install, launch, run `version`, `status`, `doctor`. | Version matches; Normal and clean. | Preserve log; run `off`; stop testing if not clean. |
| External Enable | Connect external display; Enable. | External remains main; no unnecessary virtual display under auto policy. | Restore by menu/hotkey/CLI. |
| External Restore | Restore after external Enable. | Physical display safely takes over; managed resources absent. | Retry `off`; follow Recovery Guide. |
| No-external Enable | Disconnect external displays; retain terminal; Enable. | Exact managed virtual display appears and becomes replacement. | Do not kill helper; run `off`. |
| Confirm | Confirm when prompted. | Dialog closes once; Headless becomes active. | Run Restore. |
| Immediate Restore | From Confirm Required, request Restore immediately. | Dialog stays closed and Normal returns. | Use CLI `off`. |
| Timeout rollback | Do not confirm. | Automatic rollback returns clean Normal. | Run CLI `off`. |
| Cooldown | Enable immediately after Restore, then after cooldown. | First rejected without residue; later allowed. | Inspect status/doctor. |
| Sleep/wake | In clean Normal, sleep display for 5 minutes, wake it. | Temporary unavailable may appear; no Restore Required residue; Enable works after wake. | Wake display, rerun status; Restore only if resource/recovery classified. |
| Relaunch | Relaunch in Normal and separately in Confirm Required. | State is reconciled; dialog only appears when no Restore is pending. | CLI `off`. |
| Uninstall | Restore clean Normal, then uninstall. | App/CLI removed without managed residue. | Reinstall same build and Restore if cleanup was not complete. |

Do not perform UAT during critical work. Preserve SSH access where possible.
